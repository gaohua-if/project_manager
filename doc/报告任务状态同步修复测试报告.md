# 报告任务状态同步修复测试报告

- 生成时间: `2026-07-01`
- 修复范围: `external_session_id` 状态同步、报告写回判定、同步更新竞态保护
- 结论: `PASS`

## 1. 修复结论

本轮修复后，Aida 不再只对 `external_task_id` 调用 Agent 平台状态接口；当 `ai_runs.external_task_id` 为空但 `external_session_id` 非空时，会使用同一个平台兼容接口 `GET /api/task/{external_session_id}/status` 查询状态。

对 `report_agent_run` 增加了报告写回约束：平台 session `completed` 但 Aida 没有 `business_id/report_id` 写回时，不再误判为成功，而是标记为 `failed`，错误为 `managed agent session completed without report writeback`。平台刚 completed 后保留 2 分钟写回缓冲。

| platform failed session | Aida 同步为 `failed` 并保存错误 |
| 超时 pending run | Aida 同步为 `timeout` |
