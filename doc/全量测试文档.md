- 本文共 500 条用例，编号 `FT-001` 到 `FT-500`。

- 所有写入类测试必须使用唯一前缀：`FULL-E2E-YYYYMMDD-HHmmss`。

## 7. 自动化执行流程

1. 读取 `doc/测试账号文档.md`，解析 12 个 token。
2. 计算：`TODAY`、`FROM=TODAY-14d`、`TO=TODAY`、`WEEK_START=本周一`。
3. 执行 `FT-001` 到 `FT-050`，确认账号和路由可用。
4. 创建测试夹具：
   - PM 创建小组A、小组B、多团队需求。
   - TL_A 创建小组A需求。
   - TL_B 创建小组B需求。
   - 在每个需求下创建任务，覆盖自己任务、别人任务、跨团队任务、依赖任务。
   - 每个 title 都必须包含 `AIDA_TEST_PREFIX`。
5. 执行工作台、需求看板、日报、周报用例。
6. 生成 `doc/test-reports/full-e2e-<timestamp>.md`，记录通过/失败/阻塞。
7. 清理测试数据：优先删除测试任务，再删除测试需求；删除失败保留现场并在报告中列出。

## 8. 自动化断言规则

| 场景 | 预期 |
| --- | --- |
| 缺 token/坏 token | 401 |
| 列表范围越权 | 200，但不得返回不可见数据 |
| 详情不可见 | 404 |
| 写操作无权限 | 403 |
| 参数非法 | 400 |
| 乐观锁冲突 | 409 |
| 真实不存在 | 404 |
| 报告当前记录不存在 | 404 可以接受，UI 应显示可创建/可生成入口 |
| 外部生成服务失败 | 记录为外部依赖失败，不得误判权限失败 |
