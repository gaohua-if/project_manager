# 日报周报本次开发执行范围

## 1. 文档用途

本文档用于约束本次“日报 / 周报 Agent 化改造”的开发边界，避免后续实现时把长期架构内容、Agent 平台能力、MCP 工具设计、数据库模型调整或组织权限重构一起带入本轮。

本文档不替代长期架构设计文档，也不重新定义产品方案。长期方向仍以《日报周报产品架构设计文档.md》为准；当前业务流程仍以《日报周报流程图.md》为准。本次开发只按本文档列出的范围执行。

## 3. 本次开发目标

1. 将 6 类日报 / 周报入口统一收敛为“报告内容管理弹窗”。
2. 移除旧 Skill 生成心智和所有来源手选心智。

4. 汇总报告按“可用来源”生成，不等待个人确认。
5. 保留 Agent 不可用或生成失败时的手写兜底。

## 8. 状态与操作规则

| 状态 | 展示含义 | 可操作按钮 |
| --- | --- | --- |
| 未生成 | 当前周期没有报告内容 | 智能生成、直接手写；如果 Agent 未配置则展示去配置 Agent、直接手写 |
| 生成中 | Agent 正在生成报告内容 | 可关闭弹窗或等待；不展示保存修改、确认无误 |
| AI 已生成 | Agent 已生成内容，用户尚未确认 | 确认无误、保存修改、重新生成 |
| 已确认 | 用户确认报告内容无误。个人报告可展示为“本人已确认”；小组 / 部门报告展示为“已确认” | 保存修改、重新生成 |
| 已修改 | 用户编辑并保存过内容。个人报告可展示为“本人已修改”；小组 / 部门报告展示为“已修改” | 保存修改、重新生成、确认无误 |
| 手写报告 | 用户直接手写并保存 | 保存修改、重新生成、确认无误 |
| 生成失败 | Agent 生成失败 / 超时 / 回写失败 | 重新生成、直接手写 |
| Agent 未配置 | 当前用户没有可用报告 Agent | 去配置 Agent、直接手写 |

操作规则：

1. “智能生成”只触发生成动作，不要求用户选择来源。
2. “重新生成”不要求用户重新选择来源。
3. 当前报告已有手写内容、已修改内容或已确认内容时，点击“重新生成”必须二次确认，避免误覆盖用户已经编辑或确认过的报告。
4. “直接手写”进入空编辑器或当前内容编辑态，保存后状态为手写报告或已修改。
5. “确认无误”必须依赖可落库接口；如果当前没有可复用接口支持确认状态落库，本次不要实现前端假确认，可以隐藏、置灰或标记为待接口支持。
6. 不允许出现刷新后丢失的确认状态。
7. “确认无误”只改变可信度状态，不作为汇总报告能否使用的前置条件。
8. “保存修改”保存正文编辑结果，状态变为已修改。
9. “去配置 Agent”只作为入口跳转或占位，不在本弹窗内实现 Agent 配置。

## 9. 汇总来源规则

### 9.1 可用来源

以下状态视为可用来源，可参与小组 / 部门汇总：

- AI 已生成
- 已确认
- 已修改
- 手写报告

### 9.2 缺失来源

以下状态视为缺失来源，不参与正文汇总，只进入来源摘要：

- 未生成
- 生成中
- 生成失败
- Agent 未配置且未手写

### 9.3 来源摘要

汇总报告弹窗需要展示来源摘要，但不展示来源选择控件。

来源摘要允许最小展示，必须以现有接口可获得的数据为准，不能前端 mock 或编造分布。

如果现有接口能提供完整分布，来源摘要表达：

- 可用来源数量
- 缺失来源数量
- AI 已生成但未确认数量
- 已确认数量
- 已修改数量
- 手写数量

如果现有接口只能提供部分信息，则只展示可用 / 缺失数量等可真实计算的字段。

如果现有接口无法提供来源摘要所需数据，则展示：

```text
来源摘要暂不可用
```

示例：

```text
来源摘要：8 / 10 可用，3 人 AI 已生成未确认，2 人缺失
```

### 9.4 PM 独立个人来源

PM 不属于 TL 小组，因此 PM 不进入小组日报 / 小组周报来源。

部门日报 / 部门周报的来源包含：

- 小组日报 / 小组周报
- PM 等独立个人用户的个人日报 / 个人周报

PM 独立个人来源只影响部门汇总，不影响 TL 小组汇总。
