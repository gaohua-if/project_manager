# 需求 / 任务字段校验规则

## 1. 校验原则

本规则用于需求和任务的新增、编辑接口，以及对应前端表单。

校验目标是保持输入宽松，只拦截会造成脏数据、展示破坏、数据库异常或状态污染的内容。

必须遵循：

1. 不限制中文、英文、数字、常用标点、括号、斜杠、emoji、`<`、`>`、`&` 等普通文本字符。
2. 不做敏感词、XSS、HTML 语义、URL 可访问性校验。
3. 不强制截止日期必须晚于今天。
4. 不强制任务截止日期晚于创建时间。
5. 基础字段校验失败统一返回 `400 Bad Request`。
6. 权限、存在性、跨需求依赖、循环依赖等业务规则仍按业务错误处理。
7. 错误信息至少应能定位到字段。

---

## 2. 通用规则

### 2.1 字符串 trim

所有字符串字段在校验和保存前都应去除首尾空白。

规则：

| 场景 | 规则 |
| --- | --- |
| 新增必填字段 | trim 后为空，校验失败 |
| 编辑必填字段 | 字段传入后 trim 为空，校验失败 |
| 编辑可空字段 | 字段传入后 trim 为空，按清空处理 |
| 字段未传 | 编辑时不修改 |

示例：

| 输入 | 结果 |
| --- | --- |
| `"  需求标题  "` | 保存为 `"需求标题"` |
| `"   "` | 必填字段校验失败 |
| `"\t需求标题\t"` | 保存为 `"需求标题"` |
| `"需求\t标题"` | 标题校验失败 |

### 2.2 文本字符

标题、描述、验收标准允许大部分正常业务文本。

允许：

| 类型 | 是否允许 |
| --- | --- |
| 中文 | 允许 |
| 英文 | 允许 |
| 数字 | 允许 |
| 空格 | 允许 |
| 常用标点 | 允许 |
| 括号 | 允许 |
| 斜杠 | 允许 |
| emoji | 允许 |
| `<`、`>`、`&` | 允许 |

禁止不可见控制字符：

| 范围 |
| --- |
| `U+0000` 到 `U+0008` |
| `U+000B` |
| `U+000C` |
| `U+000E` 到 `U+001F` |
| `U+007F` |

字段差异：

| 字段 | 换行 | tab |
| --- | --- | --- |
| title | 不允许 | 不允许 |
| description | 允许 | 允许 |
| acceptance_criteria | 允许 | 允许 |

### 2.3 字符长度

文本长度按 Unicode 字符数统计，不按字节数统计。

原因：

1. 中文、emoji 都是正常业务输入。
2. 按字节数统计会误伤中文和 emoji。

## 3. 通用字段规则

### 3.1 UUID 字段

以下字段必须是合法 UUID 格式：

| 字段 |
| --- |
| `requirement_id` |
| `assignee_id` |
| `team_ids[]` |
| `depends_on_ids[]` |

UUID 只校验格式，不强制 UUID version。

非法示例：

| 输入 |
| --- |
| `abc` |
| `123` |
| 空字符串 |
| 非 UUID 格式字符串 |

### 3.2 日期字段

适用字段：

| 对象 | 字段 |
| --- | --- |
| 需求 | `deadline` |
| 任务 | `due_date` |

规则：

| 场景 | 规则 |
| --- | --- |
| 未传 | 不设置或不修改 |
| 空字符串 | 清空 |
| 非空字符串 | 必须是 `YYYY-MM-DD` |
| 非真实日期 | 校验失败 |
| 早于今天 | 允许 |
| 早于创建时间 | 允许 |

合法示例：

| 输入 |
| --- |
| `2026-06-25` |
| `2026-01-01` |

非法示例：

| 输入 |
| --- |
| `2026-02-31` |
| `2026-13-01` |
| `2026-00-10` |
| `2026/01/01` |
| `2026-6-1` |

### 3.3 URL 字段

适用字段：

| 字段 |
| --- |
| `feishu_doc_url` |

规则：

| 场景 | 规则 |
| --- | --- |
| 未传 | 不设置或不修改 |
| 空字符串 | 清空 |
| 非空字符串 | 必须是 `http://` 或 `https://` URL |
| host 为空 | 校验失败 |
| 长度超过 2048 | 校验失败 |
| 非飞书域名 | 允许 |
| URL 不可访问 | 不校验 |

合法示例：

| 输入 |
| --- |
| `https://example.com/doc?id=1#section` |
| `http://example.com/doc/123` |

非法示例：

| 输入 |
| --- |
| `javascript:alert(1)` |
| `www.example.com/doc` |
| `/doc/123` |

### 3.4 枚举字段

#### 需求 status

允许值：

| 值 |
| --- |
| `todo` |
| `review` |
| `active` |
| `completed` |
| `cancelled` |

#### 需求 priority

允许值：

| 值 |
| --- |
| `low` |
| `medium` |
| `high` |
| `urgent` |

#### 任务 status

允许值：

| 值 |
| --- |
| `todo` |
| `in_progress` |
| `done` |

说明：

1. `blocked` 是派生展示状态。
2. 用户不能直接把任务状态改成 `blocked`。
3. `blocked` 不应写入任务真实状态字段。

#### 任务 priority

允许值：

| 值 |
| --- |
| `low` |
| `medium` |
| `high` |

#### 空值规则

| 场景 | 规则 |
| --- | --- |
| 新增时 priority 为空 | 默认 `medium` |
| 编辑时 priority 显式传空 | 校验失败 |
| 编辑时 status 显式传空 | 校验失败 |
| 编辑时字段未传 | 不修改 |

### 3.5 数组字段

适用字段：

| 字段 | 类型 |
| --- | --- |
| `team_ids` | ID 数组 |
| `depends_on_ids` | ID 数组 |
| `acceptance_criteria` | 文本数组 |

通用规则：

| 类型 | 规则 |
| --- | --- |
| ID 数组 | 每项 trim |
| ID 数组 | 空项校验失败 |
| ID 数组 | 每项必须是 UUID |
| ID 数组 | 重复项去重 |
| 文本数组 | 每项 trim |
| 文本数组 | 空项过滤 |
| 文本数组 | 允许清空 |

## 4. 需求新增规则

### 4.1 字段规则

| 字段 | 规则 |
| --- | --- |
| `title` | 必填；trim 后 1-120 个 Unicode 字符；不允许换行；不允许 tab；不允许不可见控制字符 |
| `description` | 必填；trim 后 1-10000 个 Unicode 字符；允许换行和 tab；不允许不可见控制字符 |
| `feishu_doc_url` | 可空；非空必须是 `http` 或 `https` URL；host 必填；最大 2048 字符 |
| `priority` | 可空；空则默认 `medium`；非空必须是 `low` / `medium` / `high` / `urgent` |
| `deadline` | 可空；非空必须是合法 `YYYY-MM-DD` 日期 |
| `team_ids` | 必填；至少 1 个；每个必须是 UUID；空项失败；去重后仍至少 1 个 |
| `acceptance_criteria` | 可空；最多 50 条；空项过滤；每条 1-1000 个 Unicode 字符；允许换行和 tab；不允许不可见控制字符 |

### 4.2 业务规则

| 规则 |
| --- |
| team 必须存在 |
| 当前用户必须有权限给指定 team 创建需求 |
| 多 team 需求必须符合当前权限规则 |

| `feishu_doc_url` | 未传不改；传空则清空；非空必须是 `http` 或 `https` URL；host 必填；最大 2048 字符 |
| `priority` | 未传不改；传入后不能为空；必须是 `low` / `medium` / `high` / `urgent` |
| `status` | 未传不改；传入后不能为空；必须是 `todo` / `review` / `active` / `completed` / `cancelled` |

## 6. 任务新增规则

### 6.1 字段规则

| 字段 | 规则 |
| --- | --- |
| `requirement_id` | 必填；必须是 UUID |
| `title` | 必填；trim 后 1-120 个 Unicode 字符；不允许换行；不允许 tab；不允许不可见控制字符 |
| `acceptance_criteria` | 可空；最多 50 条；空项过滤；每条 1-1000 个 Unicode 字符；允许换行和 tab；不允许不可见控制字符 |
| `assignee_id` | 必填；必须是 UUID |
| `priority` | 可空；空则默认 `medium`；非空必须是 `low` / `medium` / `high` |
| `due_date` | 可空；非空必须是合法 `YYYY-MM-DD` 日期 |
| `depends_on_ids` | 可空；最多 50 个；每个必须是 UUID；空项失败；重复项去重 |

### 6.2 业务规则

| 规则 |
| --- |
| requirement 必须存在 |
| 当前用户必须有权限在该 requirement 下创建任务 |
| assignee 必须存在 |
| assignee 必须属于合法指派范围 |
| depends_on_ids 对应任务必须存在 |
| depends_on_ids 对应任务必须属于同一 requirement |
| depends_on_ids 不得形成循环依赖 |
| 当前用户必须有权限引用这些依赖任务 |

## 8. 任务依赖规则

### 8.1 depends_on_ids

适用于任务新增时初始化依赖。

| 场景 | 规则 |
| --- | --- |
| 字段未传 | 不设置依赖 |
| 传空数组 | 不设置依赖 |
| 包含空字符串 | 校验失败 |
| 包含非法 UUID | 校验失败 |
| 包含重复 ID | 去重 |
| 超过 50 个 | 校验失败 |
| 引用不存在任务 | 业务错误 |
| 引用其他需求下任务 | 业务错误 |
| 形成循环依赖 | 业务错误 |

### 8.2 单个依赖新增 / 删除

适用于已存在任务维护依赖关系。

| 字段 | 规则 |
| --- | --- |
| `depends_on_id` | 必填；必须是 UUID；不能是当前任务自身 |
| `task_id` | 必填；必须是 UUID |

业务规则：

| 规则 |
| --- |
| owner task 必须存在 |
| dependency task 必须存在 |
| 两个任务必须属于同一 requirement |
| 新增依赖不得形成循环 |
| 当前用户必须有权限维护 owner task 依赖 |

## 9. 验收标准

### 9.1 文本

| 场景 | 预期 |
| --- | --- |
| title 为 `"  测试需求  "` | trim 后保存 `"测试需求"` |
| title 为空字符串 | 400 |
| title 全是空格 | 400 |
| title 包含换行 | 400 |
| title 包含 tab | 400 |
| title 包含 emoji | 允许 |
| title 包含 `<>&` | 允许 |
| title 超过 120 个 Unicode 字符 | 400 |
| description 包含换行 | 允许 |
| description 包含 tab | 允许 |
| description 超过 10000 个 Unicode 字符 | 400 |

### 9.2 UUID

| 场景 | 预期 |
| --- | --- |
| requirement_id 是合法 UUID | 允许 |
| requirement_id 是普通字符串 | 400 |
| assignee_id 是空字符串 | 400 |
| team_ids 中包含非法 UUID | 400 |
| depends_on_ids 中包含非法 UUID | 400 |
| depends_on_ids 中包含重复 ID | 去重 |

### 9.3 日期

| 场景 | 预期 |
| --- | --- |
| deadline 为 `2026-06-25` | 允许 |
| deadline 为 `2026-02-31` | 400 |
| deadline 为 `2026/06/25` | 400 |
| deadline 为空字符串 | 清空 |
| due_date 早于今天 | 允许 |

### 9.4 枚举

| 场景 | 预期 |
| --- | --- |
| 需求 status 为 `todo` | 允许 |
| 需求 status 为 `cancelled` | 允许 |
| 需求 status 为 `blocked` | 400 |
| 任务 status 为 `in_progress` | 允许 |
| 任务 status 为 `blocked` | 400 |
| priority 为空，新增需求 | 默认 `medium` |
| priority 为空，编辑需求且显式传入 | 400 |
| priority 为非法值 | 400 |

### 9.5 验收标准

| 场景 | 预期 |
| --- | --- |
| 字段未传 | 编辑时不修改 |
| 传 `[]` | 清空 |
| 传 `["", " "]` | 过滤后清空 |
| 传 `["标准1", ""]` | 保存 `["标准1"]` |
| 超过 50 条 | 400 |
| 单条超过 1000 个 Unicode 字符 | 400 |
| 单条包含换行 | 允许 |
| 单条包含 tab | 允许 |

### 9.6 任务依赖

| 场景 | 预期 |
| --- | --- |
| 新增任务时未传 depends_on_ids | 不设置依赖 |
| 新增任务时传 `[]` | 不设置依赖 |
| 包含空字符串 | 400 |
| 包含非法 UUID | 400 |
| 包含重复 UUID | 去重 |
| 包含当前任务自身 ID | 400 |
| 引用不存在任务 | 业务错误 |
| 引用其他需求下任务 | 业务错误 |
| 形成循环依赖 | 业务错误 |

## 10. 错误返回规则

字段校验失败：

| 项 | 规则 |
| --- | --- |
| HTTP 状态码 | `400 Bad Request` |
| 错误内容 | 必须能定位字段 |
| 错误数量 | 可以一次返回一个优先级最高的字段错误 |

业务规则失败：

| 场景 | 规则 |
| --- | --- |
| 无权限 | 返回权限错误 |
| 资源不存在 | 返回不存在错误 |
| 业务冲突 | 返回业务冲突错误 |
| 乐观锁冲突 | 返回编辑冲突错误 |

---

## 11. 范围边界

本期只要求校验尽量宽松且安全。

不做：

1. 敏感词校验。
2. HTML / XSS 语义校验。
3. URL 外部可访问性校验。
4. 截止日期必须大于今天。
5. 标题、描述、验收标准的字符白名单限制。
6. 字段级合并、diff、草稿恢复、实时协同。

必须保证：

1. 非法 UUID 不进入数据库转换流程。
2. 非法日期不进入日期字段。
3. 非法枚举不入库。
4. title 不包含换行和 tab。
5. description 和 acceptance_criteria 支持正常多行文本。
6. 文本长度按 Unicode 字符数统计。
7. `blocked` 不允许作为任务真实 status 写入。
8. 校验失败不返回 500。
