# 控制台 · 评审汇报

配合页面截图使用。

---

## 角色与可见模块

| 角色 | 我的报告 | 汇总报告 | Token 统计 | 关注对象 | 风险 |
|------|---------|---------|-----------|---------|------|
| 个人 | ✓ | — | 个人 | 我负责 / 关注的任务 | 我相关 |
| PM   | ✓ | — | 个人 | 我关注的需求 | 关注需求的风险 |
| TL   | ✓ | 组日报 / 组周报 | 组（含本人） | 组内需求 + 任务 | 组内 |
| 总监 | ✓ | 部门日报 / 部门周报 | 部门（按组） | 重点需求 | 高优先级 |

## 个人

```text
登录 → 控制台
  ↓
今日日报「草稿待确认」（系统已用今天 session 自动生成）
  ↓
点「确认日报草稿」→ 弹窗
  ↓
左：Markdown 草稿
右：任务进展建议（AI 根据 session 推断）
  ├─ 任务 A 建议 75% 进行中
  └─ 任务 B 建议 25% 进行中
  ↓
按需修改进度 → 标记「待同步」
  ↓
点「发送日报」
  ↓
日报发出 + 修改过的任务进度同步回写需求看板
  ↓
风险列表：「飞书发送联调 - 依赖阻塞」
  ↓
点「处理依赖」→ 跳转需求看板对应任务
```

## 报告状态与按钮

日报、周报、组报告、部门报告共用：

| 状态 | 按钮 |
|------|------|
| 待生成 | 生成报告 |
| 生成中 | 生成中（禁用） |
| 草稿待确认 | 查看草稿 + 发送 |
| 已发送 | 查看报告（只读） |
| 生成失败 | 重新生成 |
| 发送失败 | 重试发送 + 查看草稿 |

## 报告生成弹窗

**个人日报**

```text
步骤 1：选 session（系统默认勾选今天的 session，可调整）
  ↓
步骤 2：编辑草稿
  ├─ 左：Markdown 文本编辑
  └─ 右：任务进展建议（仅个人日报有）
  ↓
点「发送日报」→ 日报发出 + 任务进度回写
```

**周报 / 组报告 / 部门报告**

```text
步骤 1：确认来源
  ├─ 个人周报：本周日报 + 任务 + 风险
  ├─ 组报告：组内成员日报 + 成员提交情况
  └─ 部门报告：各组报告 + 各组提交情况
  ↓
步骤 2：编辑草稿（仅 Markdown，无任务建议）
  ↓
发送
```

## 任务进展建议（个人日报专属）

- AI 根据已选 session 推断任务进度 + 状态。
- 用户可改可不改，改完标「待同步」。
- 发送日报时，所有「待同步」项一次性回写需求看板。
- 不发送日报，修改不生效。

> 📸 截图：任务进展建议卡片 + 修改弹窗

## 风险

只两类：

```text
deadline
  ├─ 已超期
  └─ 临期（48h 内到期）

依赖阻塞
  ├─ 上游依赖未完成
  └─ 上游依赖已超期
```

排序：已超期 > 依赖阻塞 > 临期

点击 → 跳转需求看板，自动定位到任务 + 高亮风险点。

---

## 关注对象

用户主动订阅的需求 / 任务，分两类：

- 需求行：标题、负责人、deadline、依赖摘要、风险标签、变化说明
- 任务行：标题、所属需求、负责人、deadline、依赖状态、风险标签

排序：已超期 > 临期 > 依赖阻塞 > 进行中 > 其他 > 已完成

点击 → 跳转需求看板定位。

## Token 统计

| 角色 | 主体展示 |
|------|---------|
| 个人 / PM | 区间总量 + session 数 + 按日趋势 mini 柱 |
| TL | 组总量 + 成员/session/Token 三柱对比 + 我的 Token |
| 总监 | 部门总量 + 按组分布横向柱 + 我的 Token |

时间范围切换：昨天 / 近 3 天 / 近 7 天。
点「查看 Token 明细」→ 跳转 Token 明细页。

---

## 空状态

| 区域 | 文案 + 引导 |
|------|------------|
| 风险列表为空 | 「暂无需要关注的风险」+ 查看需求看板 |
| 关注列表为空 | 「暂无关注的需求或任务」+ 去需求看板关注 |
| 报告未生成 | 「报告尚未生成」+ 生成报告按钮 |
| 无 Token 数据 | 「暂无 session 上报记录」 |
