# 日报 / 周报流程图

## 1. 总览流程

```mermaid
flowchart TD
    A[打开报告弹窗] --> B{是否已有报告内容}

    B -->|有| C[展示报告内容]
    B -->|无| D{是否可智能生成}

    D -->|可生成| E[智能生成]
    D -->|不可生成| F[直接手写 / 去配置 Agent]

    E --> G[触发报告生成]
    G --> H[Agent 自动读取相关上下文]
    H --> I[生成报告内容]
    I --> J[回写报告内容]
    J --> K[状态：AI 已生成]

    F --> L[手写并保存]
    L --> M[状态：手写报告]

    C --> N[编辑保存]
    C --> O[确认无误]
    C --> E

    N --> P[状态：本人已修改]
    O --> Q[状态：本人已确认]

    K --> R[可被上级汇总]
    M --> R
    P --> R
    Q --> R

    R --> S[小组 / 部门汇总读取可用报告]
```

## 2. 六类报告弹窗主流程

适用于：

- 我的日报
- 我的周报
- 小组日报
- 小组周报
- 部门日报
- 部门周报

```mermaid
flowchart TD
    A[打开报告弹窗] --> B[识别报告类型与当前角色]

    B --> C[读取当前周期报告内容]
    B --> D[读取当前报告状态]
    B --> E[读取来源摘要]

    C --> F{是否已有报告内容}

    F -->|有内容| G[展示报告正文]
    G --> H[展示状态：AI已生成 / 本人已确认 / 本人已修改 / 手写报告]
    H --> I[用户可确认 / 编辑 / 重新生成]

    F -->|无内容| J{是否已配置对应 Agent}
    J -->|已配置| K[展示：智能生成 / 直接手写]
    J -->|未配置| L[展示：去配置 Agent / 直接手写]

    K --> M[点击智能生成]
    K --> N[点击直接手写]
    L --> O[跳转 Agent 配置]
    L --> N

    N --> P[进入空编辑器]
    P --> Q[保存为手写报告]

    I --> R[保存修改]
    I --> S[确认无误]
    I --> M

    R --> T[状态变为：本人已修改]
    S --> U[状态变为：本人已确认]
    Q --> V[状态变为：手写报告]
```

## 3. 手动智能生成流程

用户在报告弹窗内点击“智能生成”后，前端不再选择 session、日报、周报等来源，报告来源由 Agent 根据当前身份与权限自动读取。

```mermaid
flowchart TD
    A[用户点击：智能生成] --> B[触发报告生成]

    B --> C{是否有权限生成该类报告}
    C -->|无权限| D[提示无权限]
    C -->|有权限| E[进入生成中状态]

    E --> F[调用用户配置的报告 Agent]
    F --> G[Agent 自动读取相关上下文]
    G --> H[Agent 生成报告内容]
    H --> I[回写报告内容]

    I --> J{是否生成成功}

    J -->|成功| K[状态变为：AI 已生成]
    K --> L[前端展示报告正文]
    L --> M[用户可确认 / 编辑 / 重新生成]

    J -->|失败| N[状态变为：生成失败]
    N --> O[用户可重新生成 / 直接手写]
```

## 4. 定时自动生成流程

定时生成与手动生成的产品结果一致，区别只是触发来源不同。

```mermaid
flowchart TD
    A[定时任务触发] --> B[识别需要生成的报告]

    B --> C{用户是否开启自动生成}
    C -->|未开启| D[跳过]
    C -->|已开启| E{是否已配置对应 Agent}

    E -->|未配置| F[记录未配置状态]
    E -->|已配置| G[进入生成中状态]

    G --> H[调用对应报告 Agent]
    H --> I[Agent 自动读取相关上下文]
    I --> J[Agent 生成报告内容]
    J --> K[回写报告内容]

    K --> L{是否生成成功}

    L -->|成功| M[状态变为：AI 已生成]
    M --> N[该报告可被上级汇总使用]

    L -->|失败| O[状态变为：生成失败]
    O --> P[不阻断用户手写]
```

## 5. 小组 / 部门汇总来源流程

汇总报告不等待个人确认，只判断来源是否有可用报告。

```mermaid
flowchart TD
    A[生成汇总报告] --> B{汇总报告类型}

    B -->|小组日报 / 小组周报| C[读取 TL 管理的小组成员报告]
    B -->|部门日报 / 部门周报| D[读取小组报告 + 独立个人报告]

    D --> E[独立个人报告包含 PM 等非小组用户]

    C --> F[计算来源状态]
    E --> F

    F --> G{来源是否可用}

    G -->|AI 已生成| H[参与汇总]
    G -->|本人已确认| H
    G -->|本人已修改| H
    G -->|手写报告| H

    G -->|未生成| I[记为缺失]
    G -->|生成失败| I
    G -->|Agent 未配置且未手写| I

    H --> J[生成来源完整度]
    I --> J

    J --> K[展示来源摘要]
    K --> L[例如：8/10 可用，3 人 AI 未确认，2 人缺失]

    L --> M[Agent 生成汇总报告]
    M --> N[状态变为：AI 已生成]
```

## 6. 报告状态流转

```mermaid
stateDiagram-v2
    [*] --> 未生成

    未生成 --> 生成中: 点击智能生成 / 定时触发
    未生成 --> 手写报告: 直接手写并保存
    未生成 --> Agent未配置: 无可用Agent

    Agent未配置 --> 手写报告: 直接手写
    Agent未配置 --> 未生成: 配置Agent后返回

    生成中 --> AI已生成: 生成成功
    生成中 --> 生成失败: 生成失败 / 超时 / 回写失败

    生成失败 --> 生成中: 重新生成
    生成失败 --> 手写报告: 直接手写

    AI已生成 --> 本人已确认: 确认无误
    AI已生成 --> 本人已修改: 编辑保存
    AI已生成 --> 生成中: 重新生成

    本人已确认 --> 本人已修改: 再次编辑保存
    本人已修改 --> 生成中: 重新生成

    手写报告 --> 本人已修改: 再次编辑保存
    手写报告 --> 生成中: 改用智能生成
```

可用于上级汇总的状态：

- AI 已生成
- 本人已确认
- 本人已修改
- 手写报告

不可用于上级汇总的状态：

- 未生成
- 生成中
- 生成失败
- Agent 未配置且未手写

## 7. 角色与报告范围关系

```mermaid
flowchart TD
    A[用户角色] --> B[普通员工]
    A --> C[TL]
    A --> D[PM]
    A --> E[Director / Admin]

    B --> B1[我的日报]
    B --> B2[我的周报]

    C --> C1[我的日报]
    C --> C2[我的周报]
    C --> C3[小组日报]
    C --> C4[小组周报]

    D --> D1[我的日报]
    D --> D2[我的周报]
    D --> D3[不属于任何小组]
    D --> D4[不生成小组日报 / 小组周报]

    E --> E1[我的日报]
    E --> E2[我的周报]
    E --> E3[部门日报]
    E --> E4[部门周报]

    C3 --> F[来源：小组成员个人日报]
    C4 --> G[来源：小组成员个人周报]

    E3 --> H[来源：小组日报 + 独立个人日报]
    E4 --> I[来源：小组周报 + 独立个人周报]

    D1 --> H
    D2 --> I
```

## 8. 核心产品结论

1. AI 已生成就是有效报告，不等待用户确认。
2. 确认 / 修改是可信度标记，不是汇总门槛。
3. Agent 自动读取相关上下文，报告弹窗不再选择来源。
4. PM 是独立个人用户，不属于 TL 小组，但可作为部门独立个人来源参与部门汇总。
5. 六类报告弹窗统一为“报告内容管理弹窗”，不再保留 Step 流程、Skill 预设、上传 skill.md、发送对象等旧交互。
