## 一、未实现 (0%)

### 1. AI 自动生成验收标准 (PRD §5、§9.6)
- **PRD**: 创建需求时 AI 生成 3-8 条 AC,任务-AC 由 TL 手动关联。
- **现状**: `api/handler/requirement.go:132` 仅写入字符串占位 `(AI will generate acceptance criteria)`,**从未调用任何 AI**。`Update` 接口还不允许更新 `acceptance_criteria` 字段 → AC 永远是占位符。
- **影响**: §10 验收标准 "AI 自动生成验收标准" 直接未达成;且因 AC 全是占位符,任务-AC 关联无意义。

### 2. AI 自动匹配 Session→Task (PRD §5、§10)
- **PRD**: 员工上报 session 后 AI 自动关联任务,员工可确认或覆盖。
- **现状**: `BatchUpload` 时 `task_id` 永远为 NULL;`match_confidence` 字段存在但从未被写入。需用户手动 `PUT /sessions/{id}/task`。
- **影响**: §10 "员工上报 session 后 AI 自动关联任务" 未达成。

### 3. Token 多维统计 / 可视化 (PRD §5、§9.2、§9.3、§9.4)
- **PRD**: Token 按日/周/月;饼图可切 需求/任务/团队/成员/模型 维度。
- **现状**:
  - 数据存得对(`token_usage` 表粒度足够)。
  - **无任何 `/tokens` 聚合端点**,无日/周/月,无分组。
  - 仅 `daily_report` 文本里 SUM 了一下(`report.go:293`)。
  - Web Dashboard **无 Token 趋势图/饼图**,与原型中"Token 趋势 + 维度切换饼图"严重不符。
- **影响**: 总监/PM/TL/员工视图的 Token 模块全部空缺。

### 4. 周报 & 部门每日进展 (PRD §6)
- **PRD**: 个人日报/团队日报/部门每日进展/个人周报/团队周报/PM 周报/部门周报 共 7 类,周五 17:00 自动生成到飞书。
- **现状**:
  - 仅实现 **个人日报** + **团队日报**。
  - 缺失:部门每日进展、个人周报、团队周报、PM 周报、部门周报。
- **影响**: PRD §6 表格中 5/7 报告类型未实现。

### 5. 飞书集成 (PRD §4、§7、§8)
- **PRD**: 飞书 SSO + 开放 API + 文档 API(报告生成)。
- **现状**:
  - **无 OAuth2 SSO**:`auth.go:33` 仅按 name 字符串匹配,**无密码、无 token、无飞书回调**。
  - **无飞书开放 API**(PM 数据源)集成。
  - **无飞书文档 API**:报告仅存 URL 字段,需手动填写,不会自动写入飞书文档。
- **影响**: §10 "日报 20:00 / 周报周五 17:00 自动生成到飞书文档" 完全未达成。

### 6. 团队活跃度 & 沉寂预警 (PRD §2、§9.3)
- **PRD**:
  - 活跃 = 当日 ≥1 个已上报 Session。
  - Dashboard 显示 3 团队活跃度柱状图(如 11/13、9/13、10/14)。
  - 沉寂预警: 连续 3 天无 session。
- **现状**:
  - Dashboard 无 "3 团队活跃度" 卡片/柱图。
  - 无查询接口。
  - 无沉寂检测逻辑。
- **影响**: 总监视图核心指标缺失。

### 7. 跨团队阻塞升级 (PRD §3.2、§3.1 RACI)
- **PRD**: 阻塞由双方 TL 协商 → PM 协调 → 总监升级;原型显示 `T-104 跨团队阻塞`、`‼06-10 deadline 预警`。
- **现状**: 完全缺失,无 escalation 模型/状态机/通知。
- **影响**: 升级路径完全不可用。

### 8. 数据生命周期 (PRD §7)
- **PRD**: 6 个月热存储(可查询),超期归档冷存储。
- **现状**: 完全缺失,无 TTL、无归档任务。

### 9. 周报定时任务 (PRD §6)
- **PRD**: 日报每天 20:00,周报周五 17:00。
- **现状**: daemon 默认 18:00,**与 PRD 时间不符**;且需手动启动 daemon。

## 二、实现错误 (与 PRD 不符)

### 1. 后端无 RBAC (PRD §3.1)
- **问题**: `main.go:66-102` 所有路由无角色校验;任何登录用户都能改任意任务、关联任意 AC、添加任意依赖。
- **PRD 要求**: 创建需求 = 总监/PM/TL;拆任务 = TL only;关联 AC = TL only;分配任务 = TL only。
- **影响**: 越权风险,§3.1 RACI 完全落空。

### 2. 登录认证 (PRD §7、§8)
- **问题**: 飞书 SSO + TLS 1.3 要求;实际只有 name 字符串匹配,无任何安全机制。

### 3. "未勾选永不离机" 未实现 (PRD §7)
- **PRD**: 员工勾选后上报,未勾选永不离机。
- **现状**: daemon `upload --all` 直接上传所有,无勾选流程。

### 4. 任务依赖类型未暴露 (PRD §5)
- `dep_type` 字段存在,但 API 只硬编码 `finish_to_start`,无法选择 `start_to_start` 等。

## 三、实现不完善 (部分实现)

### 1. 任务/AC/进度联动 (~85%)
- **OK**: 任务关联 AC、依赖、assignee、creator_tl 校验、AC 自动完成检测、需求进度自动更新 (`task.go:135-406`、`requirement.go:235-287`)。
- **缺陷**: AC 是占位符时,进度计算分母仍把占位符计入 → 进度数据失真。

### 2. Session 上报/撤回 (~75%)
- **OK**: daemon 上报、BatchUpload 幂等、raw log 存 MinIO、撤回物理删除 (`session.go:382-412`)。
- **缺陷**: AI 匹配缺失(见 §一.2);勾选流程缺失。

### 3. 日报生成 (~40%)
- **OK**: 个人日报 + 团队日报通过 claude CLI 生成。
- **缺陷**: 时间错(18:00 vs 20:00);无周报;无飞书文档写入。

### 4. 角色权限 (~30%)
- **OK**: `users.role` CHECK 约束齐全;前端按 localStorage.role 切换视图。
- **缺陷**: 后端零校验 → 前端绕过即可越权。

### 5. Web Dashboard 与原型差距
- 原型有 4 个分角色视图(总监/TL/PM/员工)+ 流程图,当前 `dashboard/page.tsx` 仅 4 个 stat 卡 + requirements 表。
- 缺失模块:团队活跃度柱图、Token 趋势、Token 饼图、跨团队阻塞告警、沉寂预警、deadline 倒计时高亮。
